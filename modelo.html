Objetivos

Modelo OSI (Open Systems Interconnection)

Creado en 1984 por la ISO (Organización Internacional para la Estandarización) es un estándar definido para llevar a cabo las comunicaciones en la red.

establece un conjunto de estándares que aseguran que los sistemas de red sean interoperables y capaces de comunicarse entre sí.

El modelo OSI se divide en siete capas, cada una de las cuales tiene una función específica en la transmisión de datos a través de una red.

El modelo OSI es una herramienta útil para entender cómo funcionan las redes de ordenadores y cómo se comunican los dispositivos en una red. Al dividir la comunicación en siete capas, se pueden identificar claramente las funciones de cada capa y facilitar el desarrollo de sistemas de red más avanzados y eficientes.

Ventajas

Divide la comunicación de red en partes más pequeñas y sencillas.

Normaliza los componentes de red para permitir el desarrollo y el soporte de los productos de diferentes fabricantes.

Permite a los distintos tipos de hardware y software de red comunicarse entre sí.

Impide que los cambios en una capa puedan afectar las demás capas, para que se puedan desarrollar con más rapidez.

Divide la comunicación de red en partes más pequeñas para simplificar el aprendizaje.

Capas del modelo OSI

Capa 1 - Capa física:

Esta capa se encarga de la transmisión física de los datos a través de un medio de comunicación. Se define el tipo de cable, la longitud máxima, la frecuencia y el formato de la señal. Los dispositivos de esta capa incluyen cables, conectores, concentradores y repetidores.

Capa 2 - Enlace de datos:

Esta capa se encarga de la transferencia fiable de datos entre dos dispositivos conectados directamente. Se divide en dos subcapas: Control de acceso al medio (MAC) y Control de errores (LLC). Los dispositivos de esta capa incluyen dispositivos de red, como puntos de acceso inalámbricos y adaptadores de red.

Capa 3 - Red:

Esta capa se encarga de la transferencia de datos a través de una red de varios dispositivos. Se define cómo se enrutan los datos entre dispositivos y cómo se entregan de forma fiable. Los dispositivos de esta capa incluyen routers y gateways.

Capa 4 - Transporte:

Esta capa se encarga de la transferencia de datos extremo a extremo. Asegura que los datos lleguen al destino de manera fiable y en el orden correcto. Se utiliza para establecer sesiones de comunicación y para controlar el flujo de datos. Los dispositivos de esta capa incluyen puertos TCP y UDP.

Capa 5 - Sesión:

Esta capa se encarga de la gestión de sesiones de comunicación entre aplicaciones. Establece, mantiene y finaliza las sesiones. Los dispositivos de esta capa incluyen programas de mensajería instantánea y de videoconferencia.

Capa 6 - Presentación:

Esta capa se encarga de la presentación de los datos de forma que las aplicaciones puedan interpretarlos. Se encarga de la conversión de formatos de datos y la codificación de los datos para la transmisión. Los dispositivos de esta capa incluyen software de conversión de archivos y de cifrado.

Capa 7 - Aplicación:

Esta capa se encarga de la interacción directa con el usuario y de la gestión de las aplicaciones que utilizan la red. Los dispositivos de esta capa incluyen navegadores web, clientes de correo electrónico y programas de intercambio de archivos.
